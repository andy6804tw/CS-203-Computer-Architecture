Array Multiplier Online help

How to Run
  • This program multiplies the two numbers specified in the boxes A and X in 2's complement representation.
  • Two numbers can be given in either the binary (bin) or decimal (dec) number system and are 2's complement numbers. Thus, any binary number specified with a 1 in the MSB represents a negative number. Any decimal number specified can be negative.
  • The number of bits required to represent A or Xshould be specified in the Number of Bits field.
    • If a decimal number is specified, its binary representation will be sign extended to represent negative or positive numbers correctly. For example, if a -3 is entered for A or X with Number of Bits=4, the array multiplier will use 1101 to do the multiplication.
    • If a binary number is specified, it will not be sign extended when doing the multiplication. For example, if 111 is entered with Number of Bits=4, the array multiplier will use 0111 to do the multiplication.
  • A Delay calculation is made based upon the values entered in the Signal Delays section. These values represent the corresponding delays from each individual adder and are the same for all three types of adders. Since it is assumed that the A to S delay is greater than the Cin to Cout delay, the total delay will be (N-2) x AtoS + AtoCout + (N-2) x CinToCout + CinToS
    • Delay from A or X input to its Sum output (AtoS)
    • Delay from A or X input to its Carry output (AtoCout)
    • Delay from Carry input to its Carry output (CinToCout)
    • Delay from Carry input to its Sum output(CinToS)




    Format of Output
  • After entering the required values in the form, click the Compute button. This will display a diagram of the Array Multiplier (see Figure 6.19 in the textbook) representing an N x N multiplier if N is the Number of Bits.

  • Each row of the diagram consists of different types of full adders such as the one shown below:



  • There are 3 different types of full adders used, where the bubbles represent negative weights:
    • A standard full adder (typeI), has 3 inputs and 2 outputs- all positve in weight.
    • A type II adder will have a bubble on one of its inputs. It will have also have a bubble on its outgoing sum.
    • A type II' adder will have a bubble on two out of three of its inputs. It will have a bubble on its outgoing carry.
  • Timing data will also be displayed showing the total time for all of the product bits to be valid. The MSB of the product will always be the slowest and is the critical path.
