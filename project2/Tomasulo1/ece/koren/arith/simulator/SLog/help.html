Sign/Logarithm Simulator Online help


About the Simulator
This program converts the two numbers specified in boxes A and B into sign/logarithm format.
The program then performs the arithmetic calculation specified (add, subtract, etc...) using the sign/logarithm number system.
The input values A and B are first converted to sign/logarithm format according to the input parameters, k and l.
k sets the number of integer bits and l defines the number of fractional bits.
The representation of a number in sign/logarithm format is defined as the sequence of a single sign bit, s, followed by k integer bits and l fractional bits. k and l comprise the 2's complement representation of the input value. This sign bit, s, is then used to sign the value of this 2's complement number.
EA = s aa k-1a k-2a 1a 0 . a -1a -2a -l
The total number of bits in this system is therefore, s + k + l bits. It is important to note that only k bits are available to represent integer numbers. If k = 5 and a number has a "1" in the 24 position (MSB), a 2's complement result will occur which may be -24 off from the correct value. For this reason, it is recommended to set k high enough to allow for sign-extended values. NOTE: The "square" function is the most likely operation to result in overflow.
A number is converted to sign/logarithmic format by the following equation:
EX = log2 | X |
The sign of EX remains the same as the original operand, X.
To convert a number back from sign/logarithm format:
X = 2Ex
There is no representation for "0" in the sign/logarithm number system because log2 | 0 | is undefined. If "0" is entered as an operand, the value for E0 is set to all "1's." For k = 4 and l = 4, E0 = "1111.1111"

Using the Simulator

To perform a calculation in the sign/logarithm number system, enter values for A, B, k, and l. Select an operation from the list by pushing the corresponding "radio" button to the right of the description. Press the "Compute" button to calculate the operation and display the output to the this form.
Addition and subtraction are the most complicated operations to perform in the sign/logarithm number system. These operations require the result of a function, F(x), defined as:
F+( EA - EB ) = log2 | 1 + 2 - ( EA - EB ) |
For addition and...
F-( EA - EB ) = log2 | 1 - 2 - ( EA - EB ) |
...for subtraction. To speed calculations for addition and subtraction, values for F(x) are generally stored in ROMs. To view the F+ or F- ROMs, enter values for A, B, k, and l and then click the "Show ROM PHI+" or "Show ROM PHI-" buttons.


About the Arithmetic Operations

Addition in the sign/logarithm number system is defined as...
Es = EA + F+( EA - EB)
for | EA | > | EB | and...
Es = EB + F+( EB - EA)
for | EB | > | EA |
Subtraction in the sign/logarithm number system is defined as...
Ed = EA + F-( EA - EB)
for | EA | > | EB | and...
Ed = EB + F-( EB - EA)
for | EB | > | EA |
Multiplication is reduced to the addition of two numbers in sign/logarithm format...
Ep = EA + EB
Division is the subtraction of two numbers in sign/logarithm format...
Eq = EA - EB
The Reciprocal of a numbers is calculated in the sign/logarithm number system by...
EA-1 = 2 - Ea
Square Root may be calculated as...
EA1/2 = 2 Ea / 2
Square of a number is found by...
EA2 = 2 2 * Ea
For more information on the sign/logarithm number system, please see Chapter 10 of "Computer Arithmetic Algorithms" by Israel Koren.
