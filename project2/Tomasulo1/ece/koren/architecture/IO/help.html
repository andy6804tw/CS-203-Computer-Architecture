Help on how to use I/O system evaluation tool

This tool is to help you to know how to run the program and to understand the process of designing an I/O system that meets the goals for cost and performance.

The tool has five main evaluation buttons. These are:-

  • Reset Values
    • This button resets the inputs to default values set by the program BUT it does not evaluate these input to produce any outputs. The user can change any of the inputs then click on the evaluation buttons to produce an output. An error message will pop up if an input is left blank or if there is an invalid data entry. For example, if the size of the small disk is greater than the size of the large disk or if the utilization of resources is greater than 100% then an error message will pop up.
  • Specs. of Organization. This button will produce a table that describes the specifications of the the I/O system. The I/O performance is limited by the weakest link in the chain, so we evaluate the maximum performance of each link in the I/O chain for each organization to determine the maximum performance of that organization. The contents of this table gives the evaluations as follows:-

    i) Maximum IOPS for CPU

    The CPU I/O performance is determined by the speed of the CPU and the number of instructions to perform a disk I/O

    Max IOPS for CPU = (CPU MIPS) / (Number of instructions per I/O)

    ii) Maximum IOPS for main memory

    The maximum performance of the main memory is determined by the memory cycle time, the width of the main memory, and the size of the I/O transfers.

    Max IOPS for main memory =( (1/memory cycle time) * memory width ) / (size of I/O transfers)

    iii) Maximum IOPS for the I/O bus

    The I/O bus maximum performance is limited by the bus bandwidth and the size of the I/O

    Max IOPS for I/O bus = (Bus bandwidth) / (size of the I/O)

    iv) Maximum IOPS per SCSI controller

    The maximum performance for each SCSI controller is determined by the SCSI controller overhead and the SCSI bus transfer time. The whole system organization will require several controllers.

    Max IOPS per SCSI controller = 1 / (controller overhead + SCSI bus transfer time)

    But:-

    SCSI bus transfer time = (size of I/O) / (SCSI bus transfer rate)

    v) Maximum IOPS (using average seeks) for disks

    The maximum disk performance is determined by the average disk I/O time

    Max IOPS per disk = 1 / (I/O time)

    But to find the average disk access time:-

    I/O time = average seek time + average rotational latency + disk transfer time

    Average rotational latency = (0.5 rotation) / (Disk rotation speed)

    Disk transfer time = (Average I/O size) / (Disk transfer rate)

    Therefore:-

    Max IOPS for large disks = number of large disks required * max IOPS per disk

    Number of large disks required = (Desired total disk capacity) /(size of each large disk )

    Max IOPS for small disks = number of small disks required * max IOPS per disk

    Number of small disks required = (Desired total disk capacity) /(size of each small disk )

    vi) Maximum IOPS for SCSI strings

    After determining the performance of each link of the I/O chain as described above, the designer has to determine the number of SCSI buses and controllers to use and how many disks to connect to each controller.

    The I/O bus is limited by the number of SCSI controllers and the number of disks per SCSI string.

    Minimum number of strings for large disks = ceiling value ( (number of large disks required)/(maximum number of disks per SCSI string) )

    Minimum number of strings for small disks = ceiling value ( (number of small disks required)/(maximum number of disks per SCSI string) )

    The number of disks per SCSI string at full bandwidth = floor value( (maximum IOPS per SCSI controller)/(maximum IOPS per disk) )

    Ideal number of strings for large disks = ceiling value ( (number of large disks required)/(number of disks per SCSI string at full bandwidth) )

    Ideal number of strings for small disks = ceiling value ( (number of small disks required)/(number of disks per SCSI string at full bandwidth) )

    Therefore:-

    Max IOPS for SCSI string of large disks = number of strings for large disks * max IOPS per SCSI controller

    Max IOPS for SCSI string of small disks = number of strings for small disks * max IOPS per SCSI controller

  • Maximum IOPS
    • This button produces a table that summarizes the maximum IOPS ( I/O per second)for each I/O system organization. The first two columns show the IOPS when there is no restriction on the utilization of resources ie. it assumes 100% utilization of resources. The rest of the columns show the IOPS when the rules of thumb have been applied to restrict the utilization of resources.
  • Cost of Organization
    • This button produces a table showing the cost of each system organization.

    Cost of organization = CPU cost + (no. of strings * cost of each SCSI controller) + (no. of disks * disk size * cost of disk per GB) + (cost of enclosure * ceiling value(no. of disks / max. no. of disks per package) )

    The IOPS shown in the table is the minimum IOPS in each organization.

    The maximum performance of each organization is limited by the bottleneck IOPS

    Therefore:-

    IOPS = minimum IOPS (CPU, main memory, I/O bus, disk, SCSI string)

  • Resource utilization
    • This button produces a table showing the percentage of utilization of each resource before and after using the rules of thumb.

    The IOPS shown in the table is the minimum IOPS in each organization. The maximum performance of each organization is limited by the bottleneck IOPS

    Therefore:-

    IOPS for the organization = minimum IOPS (CPU, main memory, I/O bus, disk, SCSI string)

    and the resource utilization are as follows:-

    CPU utilization = (IOPS for the organization)/(maximum IOPS for CPU)

    Main memory utilization = (IOPS for the organization)/(maximum IOPS for main memory)

    I/O bus utilization = (IOPS for the organization)/(maximum IOPS for I/O bus)

    Disk utilization = (IOPS for the organization)/(maximum IOPS per disk100% utilization * number of disks)

    SCSI bus utilization = (IOPS for the organization)/(maximum IOPS per SCSI controller100% utilization * number of SCSI strings)

    seek utilization = (Average seek time)/(Time between I/Os)

    Time between I/Os = 1 / (maximum IOPS per disk)
